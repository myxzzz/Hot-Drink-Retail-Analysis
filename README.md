## 🎯 核心课题：建立基于空间属性的“门店抗冻性”及销量预估模型

因为真实的门店销量属于企业机密不可获取，本项目通过**特征工程 (Feature Engineering)** 与多维数据交叉，构建替代性的**「预估销量抗冻指数」**，以此训练和展示完整的数据清洗、建模与商业可视化能力。

### 1. 业务逻辑 (Business Logic)

* **核心假设 (Hypothesis)**：门店的选址属性决定了其在极端天气下的商业韧性。当气温发生断崖式下跌时，虽然宏观热饮需求整体上升，但“商务区（室内刚需）”门店的订单表现将远比“开放型景区（室外随机）”门店更稳定。
* **特征评估维度**：
  1. 🌡️ **气象打击因子**：历史最低气温、骤降幅度。
  2. 📈 **宏观需求增量**：特定时期“奶茶/热饮”全网搜索指数。
  3. 🏢 **地理抗性特征**：门店周边 500m 的 POI 分布（“写字楼密度” vs “公园景区密度”）。

### 2. 技术主线与学习目标 (Tech Stack & Goals)

本项目旨在全面锻炼以下核心数据分析技能：

* 🧹 **数据清洗与特征工程 (Python / Pandas)**：
  * 处理高德 API 和气象库获取的非结构化/脏数据。
  * 根据空间 POI 密度，为北京成百上千家饮品店打上“商务型/休闲型”的分类标签（特征工程）。
* 🔗 **数据规整与表连接 (SQL / Pandas Merge)**：
  * 将 `时间维度表 (气象+搜索指数)` 与 `空间维度表 (门店特征)` 进行多键连接 (Joins)。
  * 通过自定义函数计算融合指数：`预估抗冻表现 = 商务权重 × 需求增量 - 休闲权重 × 天气打击`。
* 📊 **商业智能可视化 (Power BI)**：
  * 放弃简单的 Python 静态绘图，将最终清洗好的宽表 (Flat Table) 导入 Power BI。
  * 制作包含「地图钻取、日期筛选器、极端天气折线图」的交互式商业数据看板。
